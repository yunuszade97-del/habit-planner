# Product Requirements Document (PRD)
**Project Name:** Habit Planner MVP

## 1. Обзор продукта
Habit Planner — это простое и быстрое веб-приложение для отслеживания ежедневных привычек и достижения долгосрочных целей на год. Приложение ориентировано на мобильный опыт (mobile-first дизайн) и наглядную инфографику.

## 2. Основной функционал (Реализован)
- **Аутентификация пользователей**:
  - Регистрация и вход по Email/Паролю.
  - Социальная авторизация: Google Auth.
  - Социальная авторизация: Apple Auth. (Используется официальный логотип Apple).
  - Строгая привязка к авторизации (без локального Skip-режима).
- **Управление ежедневными привычками**: Добавление, редактирование, удаление привычек.
- **Цели на год**: Создание глобальных целей с привязкой конкретных привычек-шагов к ним.
- **Отслеживание прогресса**: Интерактивная 5-дневная история выполнения прямо на карточках привычек. Расчет "strikes" (дней подряд).
- **Инфографика и статистика**:
  - Подробные модальные окна со статистикой для каждой привычки и сводной статистикой для целей на год (с круговыми диаграммами прогресса).
- **Синхронизация данных (Cloud Data Sync / База данных)**: Подключен Cloud Firestore. Привычки каждого пользователя хранятся изолированно (`users/{userId}/tasks`) и автоматически синхронизируются между устройствами в реальном времени. Локальное хранение (`localStorage`) было полностью удалено.
- **Премиум-версия (Premium/Pro)**: Реализована базовая логика триала (7 дней после регистрации в Firebase). При истечении срока аккаунт получает статус `isPremiumExpired`.

## 3. Планируемый функционал

### Фаза 3 — В разработке
- **Строгий Премиум-лок (Read-only)**: При истечении 7-дневного триала приложение полностью блокирует взаимодействие (нельзя ставить галочки, удалять, редактировать, добавлять). Интерфейс становится визуально заблокированным (`opacity`, `not-allowed`).
- **Премиум-Фоны**: Возможность смены фона/темы приложения. Выбор премиального фона при истекшем триале вызывает окно оплаты (Paywall).
- **Paywall UI**: Красивое модальное окно вместо системного `alert()` с описанием преимуществ Premium.
- **Компактная Система Заметок**: Интеграция текстовых заметок, реализованная в виде компактной UI-кнопки (эмодзи 📝), чтобы не создавать лишних визуальных вкладок.

### Фаза 4 — Запланировано
- **Биллинг и Оплата**: Подключение реальной платежной системы (Stripe/Robokassa) для активации Премиума после истечения 7-дневного триала.
- **Вебхуки**: Обновление статуса пользователя через Firebase Cloud Functions.
- **Управление подпиской**: Отображение статуса и даты истечения.

### Фаза 5 — Запланировано
- **Дополнительные настройки профиля**: Управление аккаунтом (смена пароля, удаление аккаунта по GDPR).
- **Аналитика**: Firebase Analytics + Crashlytics.
- **Онбординг**: Обучающие слайды при первом входе.
- **PWA** (опционально): Установка веб-приложения на рабочий стол телефона.

## 4. Архитектура и технологии
- **Frontend**: Vanilla HTML5, CSS3, JavaScript (ES6+). Фреймворки не используются.
- **Backend/Auth/DB**: Firebase (Authentication + Cloud Firestore).
- **Безопасность**: XSS-санитизация через `SecurityService.sanitizeHTML()`.
- **Структура файлов**:
  - `index.html` — разметка и подключение скриптов
  - `app.js` — контроллер, состояние (AppState), рендеринг
  - `auth.js` — аутентификация Firebase, управление экраном входа
  - `db.js` — CRUD операции с Firestore
  - `security.js` — санитизация пользовательского ввода
  - `styles.css` — все стили приложения

## 5. Известные ограничения
- Все ID сущностей — строки (приведение через `String()`). Это критично для совместимости с Firestore.
- Firestore Security Rules необходимо настроить для production (см. ROADMAP.md).
- Apple Auth требует Apple Developer Account и настройки Sign in with Apple.
