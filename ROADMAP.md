## Важные технические замечания для ИИ-агентов

### Архитектура проекта
- **Стек:** Vanilla JS (ES6+), HTML5, CSS3, Firebase (Auth + Firestore)
- **Точка входа:** `index.html` загружает скрипты в порядке: `security.js` → `db.js` → `auth.js` → `app.js`
- **Поток данных:** `auth.js` (логин) → `db.js` (загрузка данных) → `app.js` (рендер UI)
- **Состояние:** Глобальный объект `AppState` в `app.js`
- **Безопасность:** Все пользовательские строки проходят через `SecurityService.sanitizeHTML()` перед вставкой в DOM

### Правила при разработке
1. **Все ID — строки.** При создании новых сущностей использовать `String(Date.now())`. Сравнения ID — через `String(a) === String(b)`.
2. **Нет фреймворков.** Весь UI строится через `innerHTML` и template literals.
3. **Mobile-first.** Все стили адаптированы под `max-width: 480px`.
4. **Firebase SDK** загружается из CDN (`gstatic.com`), версия `11.4.0`.
5. **Модули** `db.js` и `auth.js` используют ES modules (`type="module"`), `app.js` и `security.js` — обычные скрипты.
6. **XSS-защита:** Обязательно использовать `SecurityService.sanitizeHTML()` для всех пользовательских данных перед рендером.

### Firestore Security Rules (КРИТИЧНО — нужно настроить)
Текущие правила Firestore должны быть обновлены для безопасности:
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /tasks/{taskId} {
      allow read, write: if request.auth != null && request.auth.uid == resource.data.userId;
      allow create: if request.auth != null && request.auth.uid == request.resource.data.userId;
    }
    match /goals/{goalId} {
      allow read, write: if request.auth != null && request.auth.uid == resource.data.userId;
      allow create: if request.auth != null && request.auth.uid == request.resource.data.userId;
    }
  }
}
```
